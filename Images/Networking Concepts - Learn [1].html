What is the purpose of placing the database server in a private subnet instead of a public subnet?
01:43 PM
Dr. Newton
Placing the database server in a private subnet enhances security by isolating it from direct internet access. A private subnet restricts inbound traffic from the internet, ensuring the database is only accessible from authorized internal resources, such as web servers within the VPC. This approach protects sensitive data by preventing unauthorized external connections. Database connectivity is controlled through security group rules that permit access only from specific resources on designated ports, such as port 3306 for MySQL. This architecture follows the principle of least privilege, allowing only necessary communication paths while blocking all other internet-bound traffic.
